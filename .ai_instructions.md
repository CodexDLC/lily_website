# Инструкции для ИИ (AI Instructions)

Этот файл содержит правила взаимодействия с ИИ-ассистентом для проекта Lily Beauty Salon.

## 1. Язык и коммуникация
- **Язык общения:** Всегда использовать РУССКИЙ язык для всех ответов в чате и при создании артефактов.
- **Артефакты:** Файлы `task.md`, `implementation_plan.md` и `walkthrough.md` должны вестись на русском языке.

## 2. Работа с Git
- **Запрет на самостоятельные действия:** ИИ запрещено самостоятельно выполнять команды `git commit` и `git push`.
- **Роль:** ИИ подготавливает код, проверяет его, но коммит и отправку на сервер выполняет только ПОЛЬЗОВАТЕЛЬ.
- **Git Add:** Использовать с осторожностью, лучше подготавливать список изменений для пользователя.

## 3. Безопасность и .gitignore
- **Доступ к скрытым файлам:** По умолчанию не заглядывать в файлы, перечисленные в `.gitignore` (особенно `.env`), без явной просьбы пользователя.
- **Переменные окружения:** Если нужна помощь с `.env`, спрашивай разрешение или предлагай использовать `.env.example`.

## 4. Контекст проекта
- **Структура проекта:** Использовать файл `project_structure.txt` в корне проекта для быстрого понимания архитектуры. Не сканируй весь проект без необходимости, если структура уже описана.
- **Пути:** Корень проекта находится на два уровня выше папки `tools/dev/`.

## 5. Инструменты разработки
- **djlint:** Использовать `djlint` для форматирования и проверки Django-шаблонов (`profile="django"`).
- **Poetry:** Все зависимости устанавливаются через `poetry add`.
