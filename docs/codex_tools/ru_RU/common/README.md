# 🛠️ Общие Утилиты (Common)

[⬅️ Назад в корень](../../README.md)

Папка `common` собирает в себе "швейцарские ножи" — функции и классы, которые не относятся напрямую к бизнес-логике проекта, но без которых не обходится ни один серьезный сервис. Они обобщены, чтобы не зависеть от логики конкретного фреймворка.

## 🏗️ Почему они здесь?
Часто разработчики пишут парсер телефонов или настройку логгера прямо в папке `core/` или `utils/` внутри Django-приложения. Это делает код "прибитым" гвоздями к фреймворку.

Размещение таких вещей в `codex_tools/common` преследует цель создать **«Базовый слой» (Base Layer)**.

## 🗂️ Включенные Утилиты

### `logger.py`
Безальтернативный выбор в пользу `loguru`. Этот файл решает классическую проблему питона: как заставить все сторонние библиотеки, использующие дефолтный модуль `logging`, писать красивые цветные логи с ротацией в файлы без настройки огромных словарей `LOGGING` в `settings.py`.
- Подключает стандартный вывод `logging` к `loguru` через `InterceptHandler`.
- Настраивает цветные логи в stdout.
- Автоматически обрабатывает JSON файлы с ошибками и zip-ротацию.
- Глушит шумные логи библиотек (например, `django.db.backends`, которые спамят SQL-запросами).

### `arq_client.py`
Зачастую требуется поставить задачу из обычного синхронного view Django в асинхронную очередь Redis.
Предоставляет `BaseArqClient`, абстрактный класс (producer) для асинхронной очереди задач `arq`.
- Встроенная обёртка создает event-loop на лету и прячет всю сложность работы с ARQ пулами.
- Позволяет отправлять задачи в пул Redis из синхронного кода.

### `cache.py`
Абстракции интерфейсов для базовых сценариев кэширования. В настоящее время это в основном структура сигнатур, готовая к подключению к стандартным механизмам кэширования в памяти или redis в зависимости от конкретного развертывания.

### `phone.py` & `text.py`
Чистые функции валидации и нормализации данных без использования ORM валидаторов.
- **`phone.py`**: Простой и надежный нормализатор телефонов (`normalize_phone`), который удаляет лишние символы и преобразует локальные немецкие номера (`0151...`) в международный формат (`49151...`).
- **`text.py`**: Инструменты для нормализации имен (например, капитализация имен через дефис) и удаления невидимых артефактов пробелов.
