# 🗓️ Модуль Календаря (Codex Calendar)

[⬅️ Назад в корень](../../README.md)

Модуль `codex_calendar` предоставляет универсальный, независимый от фреймворка генератор сетки календаря. Он решает тривиальную, но часто повторяющуюся задачу построения матрицы дней текущего месяца для отображения на UI или в Telegram-ботах.

## Зачем он нужен отдельно?
Практика показывает, что генерация сетки календаря (с учетом первых дней недели, буферных пустых ячеек в начале/конце месяца, и выходных/праздничных дней) часто пишется с нуля в каждом проекте.

Движок `CalendarEngine` выносит эту логику в единую зависимость.
Главное архитектурное решение здесь заключается в том, что календарь возвращает **универсальный словарь состояний** для каждого дня (`available`, `disabled`, `holiday`), который любой фронтенд-клиент может отрендерить в своем стиле.

## `CalendarEngine`

Сердцем модуля является `CalendarEngine`, расположенный в `engine.py`. Его цель — выводить надежные представления месяца для интерфейсов веб-сайтов, Telegram-ботов и мобильных приложений.

### Ключевые Особенности
1. **Целостные Массивы (Holistic Arrays)**: Итерируется по полным визуальным неделям (`calendar.Calendar`), заполняя дни вне текущего месяца пустыми пространствами (padding) для сохранения выравнивания матрицы.
2. **Разрешение Состояний (State Resolution)**: Помечает каждый день его состоянием: `available` (доступен), `disabled` (прошедшие дни или конкретные нерабочие дни), `holiday` (праздник) или `active` (активен).
3. **Поддержка Локалей (Locale Support)**: Интегрируется с Python пакетом `holidays` для внедрения динамических государственных праздников в матрицу.

### Пример Использования
```python
from codex_tools.codex_calendar.engine import CalendarEngine
from datetime import date

engine = CalendarEngine()
matrix = engine.get_month_matrix(
    year=2024,
    month=5,
    today=date.today(),
    holidays_subdiv="ST" # код регионального праздника (например, Германия -> Саксония-Анхальт)
)
```
