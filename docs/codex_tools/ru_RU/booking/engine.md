# ⚙️ Ядро Движка Бронирования (Booking Engine Core)

[⬅️ Назад](./README.md) | [🏠 Корень Документации](../../README.md)

Основная логика находится в `codex_tools.booking`. Она использует парадигмы скользящего окна (sliding-window) для анализа свободного времени в расписаниях.

## `SlotCalculator`

Выделенные чистые функции для временны́х вычислений.
- `merge_free_windows`: Вычитает занятые интервалы (`busy_intervals`) и перерывы (`break_intervals`) из рабочих часов мастера (`work_hours`), возвращая список непрерывных свободных окон (`free_windows`).
- `find_slots_in_window`: Принимая начало окна (`window_start`), конец окна (`window_end`), длительность (`duration_minutes`) и шаг сетки (`step_minutes`, по умолчанию 30 минут), вычисляет все валидные стартовые точки на сетке расписания.

## `ChainFinder`

Оркестратор. Он выполняет алгоритм поиска с возвратом (backtracking), чтобы преобразовать запрос `[Услуга_A, Услуга_B]` в конкретные слоты `[(Мастер_1, 10:00), (Мастер_1, 11:30)]`.

### Поток выполнения алгоритма:
1. Валидирует входящий запрос `BookingEngineRequest`.
2. Получает словарь `dict[master_id: MasterAvailability]` (рассчитанный адаптером).
3. При обработке одного дня (single day):
   - Для каждой услуги в цепочке пытается привязать её к свободному окну мастера.
   - Помещает предварительный (tentative) слот во временный стек (`_SlotCandidate`).
   - Если привязка не удалась, извлекает слот (pop) и пробует следующий доступный слот или мастера.
   - Сохраняет успешно завершенные цепочки как `BookingChainSolution`.
4. Возвращает объект `EngineResult`: сопоставленный, отсортированный и, при необходимости, сгруппированный по времени начала.

**Характеристики производительности:**
Движок обходит тяжелые валидации Pydantic внутри метода `backtrack()`, используя легковесные классы-кандидаты с `__slots__`. Он динамически масштабируется до `10,000` комбинаций с субмиллисекундной задержкой.
