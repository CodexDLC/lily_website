# 📅 Модуль Бронирования (Booking)

[⬅️ Назад в корень](../../README.md)

Этот модуль инкапсулирует бизнес-логику для вычисления свободных слотов и оптимизации бронирования для одной или нескольких услуг (цепочек). Он работает строго на обычных структурах данных Python, избегая любой жесткой привязки к ORM.

## 🗺️ Карта модуля

| Компонент | Описание |
|:---|:---|
| **[📄 Data Transfer Objects (DTO)](./dto.md)** | `frozen=True` Pydantic схемы, определяющие входные и выходные данные. |
| **[📄 Ядро (Core Engine)](./engine.md)** | Алгоритмы `ChainFinder` и `SlotCalculator`. |
| **[📄 Интерфейсы и Адаптеры (Interfaces & Adapters)](./interfaces_and_adapters.md)** | Как этот модуль связывается с внешними базами данных, такими как Django. |

## Краткий Обзор

Движок бронирования спроектирован для ответа на главный вопрос: *"Когда мастер доступен для оказания услуги X, и если есть несколько услуг в цепочке, как они стыкуются вместе?"*

Он поддерживает:
1. Поиск для одной услуги.
2. Последовательные цепочки, выполняемые одним мастером.
3. Цепочки, где услуги могут выполняться параллельно или перекрываться.
4. Перерывы, динамическое минимальное время начала (например, "не раньше, чем через 2 часа от текущего момента") и переменные паузы (буферы) между сеансами.
