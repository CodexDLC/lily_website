# Техническая база знаний и принципы разработки LILY Beauty Salon

Этот документ — основной технический стандарт проекта. Он содержит исчерпывающую информацию об архитектуре, SEO, локализации и производительности, необходимую для разработки и поддержки сайта.

---

## 1. Локализация и i18n (Архитектура «Смысловых ключей»)

Мы используем гибридную систему: интерфейсные строки в `.po` файлах и маркетинговый контент в БД.

### 1.1. Система ключей (UI)
Используется через `{% trans "key" %}`. **Запрещено** вставлять чистый текст прямо в тег `trans`.

#### Группа: Навигация (`nav_`)
| Ключ | Контекст |
| :--- | :--- |
| `nav_main_home` | Главная страница |
| `nav_main_services` | Услуги |
| `nav_main_team` | Команда |
| `nav_main_contacts` | Контакты |
| `nav_footer_all_contacts` | Ссылка "Все контакты" в футере |
| `nav_legal_impressum` | Impressum |
| `nav_legal_privacy` | Datenschutz |

#### Группа: Кнопки и Действия (`btn_`)
| Ключ | Контекст |
| :--- | :--- |
| `btn_cta_booking` | Основная кнопка записи |
| `btn_action_more` | "Подробнее" |
| `btn_form_submit` | Отправка форм |
| `btn_form_cancel` | Отмена |
| `btn_ui_toggle_footer` | Переключатель футера |

#### Группа: Услуги (`service_`)
| Ключ | Категория/Услуга |
| :--- | :--- |
| `service_cat_hair` | Прядь / Волосы |
| `service_cat_nails` | Ногти |
| `service_item_manicure` | Маникюр |
| `service_item_coloring` | Окрашивание |

### 1.2. Контент в БД (StaticTranslation)
Для текстов, которые часто меняются (Hero-секции, описания услуг), используется модель `StaticTranslation`.
- **Доступ в шаблоне**: `{{ content.home_hero_title }}`.
- **Поддержка языков**: Поля `text_de`, `text_en`, `text_ru`, `text_uk` (через `modeltranslation`).

## 2. SEO и AI-Оптимизация (AIO)

Сайт спроектирован для "человеческого" поиска (Google) и "машинного" (LLM/AI).

### 2.1. AI-Контекст (Special Purpose)
Файлы `llms_*.txt` обеспечивают AI-агентов (ChatGPT, Perplexity) точной информацией.
- **Локация**: `src/backend_django/templates/llms_[lang].txt`
- **Содержание**: Структура услуг, FAQ, методы оплаты, парковки и т.д.

### 2.2. Техническое SEO и Sitemap
- **Dynamic Sitemap**: XML генерируется через `core/sitemaps.py`. Включает динамические страницы услуг.
- **Hreflang**: Автоматическая генерация в `includes/_hreflang_tags.html`.
- **Canonicals**: Всегда указывают на основной домен, заданный в `CANONICAL_DOMAIN`.

### 2.3. Микроразметка (JSON-LD)
В файле `_schema_local_business.html` реализована схема `LocalBusiness` с поддержкой:
- **OpeningHours**: Динамически из БД.
- **GeoCoordinates**: Для карт.
- **OfferCatalog**: Глубокая вложенность категорий и услуг для Google Search.

## 3. Производительность и сборка

### 3.1. Оптимизация загрузки (Render-Blocking)
- **Critical CSS**: Инлайнится в теге `<head>` через `_critical_css.html`. Содержит стили хедера и Hero-секции.
- **Async JS/CSS**: Основные файлы подгружаются с `defer` или через preload-полифиллы.
- **Fetch Priority**: Критичные изображения (лого, первый баннер) имеют атрибут `fetchpriority="high"`.

### 3.2. Изображения (Media)
- **WebP**: Автоматическая конверсия всех Uploads.
- **Lazy Loading**: Для всех изображений вне первого экрана.

### 3.3. HTMX Интеграция
Используется для:
- Динамического фильтра услуг.
- Пошагового визарда бронирования (`booking_wizard`).
- Подгрузки контента без перезагрузки всей страницы (спасибо `hx-target`).

## 4. Конфигурация (SiteSettings)
Центральный пульт управления сайтом в Django Admin:
- **Socials**: Ссылки на Instagram, WhatsApp, Facebook.
- **Analytics**: GA4 и GTM контейнеры.
- **Service Data**: Пороги цен, время работы, координаты.
