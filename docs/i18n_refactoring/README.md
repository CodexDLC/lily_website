# Рефакторинг i18n: Переход на смысловые ключи

## Идея
Заменить длинные текстовые строки в шаблонах Django на короткие, структурированные ключи (переменные). Это позволит:
1. Уменьшить размер `.po` файлов.
2. Избежать дублирования переводов для одинаковых по смыслу элементов.
3. Упростить поддержку и локализацию.

## Структура ключа
`[группа]_[подгруппа]_[название]`
Пример: `nav_main_home`, `btn_cta_booking`.

## Группы ключей
- [Навигация (Navigation)](group_navigation.md) — Меню, ссылки в хедере и мобильном меню.
- [Кнопки и действия (Buttons & Actions)](group_buttons.md) — CTA, ссылки "подробнее", формы.
- [Футер и юридическая информация (Footer & Legal)](group_footer.md) — Ссылки в подвале, копирайты.
- [Услуги и категории (Services)](group_services.md) — Названия основных направлений.

## Процесс миграции
1. Описать маппинг в соответствующих `.md` файлах.
2. Заменить теги в шаблонах: `{% trans "Текст" %}` -> `{% trans "key_name" %}`.
3. Выполнить `python manage.py makemessages`.
4. Перенести старые переводы в новые блоки `msgstr` в `.po` файлах.
